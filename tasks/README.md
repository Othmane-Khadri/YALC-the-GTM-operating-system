# Tasks Folder

This is the handoff mechanism between the CTO session and developer/QA sessions.

## File Conventions
- `day-XX.md` — developer brief for Day XX (written by CTO, read by developer agent)
- `day-XX-report.md` — developer report for Day XX (written by developer agent, read by CTO)
- `qa-batch-X.md` — QA brief (written by CTO, read by QA agent)
- `qa-batch-X-report.md` — QA findings (written by QA agent, read by CTO)
- `fix-XX.md` — bug fix brief (written by CTO, read by fix agent)
